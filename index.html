what is?

  • Trygve Reenskaug created MVC while working as a visiting scientist at the Xerox in 1979.
  • He wanted a pattern that could be used to structure any program where users interact with a large data set.
  • His design initially had four parts: Model, View, Thing, and Editor.But after discussing with the other developers, pattern took the form: Model, View, and Controller.

What is the MVC Model?

  • A static HTML page cannot respond to user actions. Two-way interaction requires dynamic web pages. MVC is the key to understanding dynamic web application development.
  • “M” “V” “C” stands for “MODEL” “VIEW” “CONTROLLER” It's a way of organizing code that separates blocks for different tasks.

How it works?

  • The user through [view] sends a request to the controller.
  • The controller processes the request and update the [model].
  • The [model] processes the update and sends a notify to [controller].
  • The [controller] processes the notify of the [model] and displays the result on the [view].
  • The [user] wants to transfer money from one account to another account
  • He communicates with a bank employee or a payment machine[view]
  • Only the bank's internal structure[model] can to transfer money
  • Bank employee or a payment machine don't know anything about the bank's internals.
  • Controllers accept commands from the user, process them and pass the necessary instructions to the model.
  • Only the controller knows how to handle the client's request. How much money to send, the method of sending and gives the necessary commands to the internal device of the bank
